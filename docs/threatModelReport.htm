1. Spoofing the Human User External Entity  [State: Mitigation Implemented]  [Priority: High] 

Category:Spoofing
Description:Human User may be spoofed by an attacker and this may lead to unauthorized access to Web Application. Consider using a standard authentication mechanism to identify the external entity.
Justification:Our application can be used by anyone, so there is no need to authenticate persons. The program does not store or process either important and confidential information that can be somewhere or somehow used.

2. Cross Site Scripting  [State: Not Applicable]  [Priority: High] 

Category:Tampering
Description:The web server 'Web Application' could be a subject to a cross-site scripting attack because it does not sanitize untrusted input.
Justification:CORS have to be enabled because it provides access for any users to use our application.

3. Elevation Using Impersonation  [State: Mitigation Implemented]  [Priority: High] 

Category:Elevation Of Privilege
Description:Web Application may be able to impersonate the context of Human User in order to gain additional privilege.
Justification:The application gives you the only public or private link to the snippet. The application does not produce them. It only transfers link to the client.

Interaction: HTTPS Response

4. Spoofing the External Web Service External Entity  [State: Needs Investigation]  [Priority: High] 

Category:Spoofing
Description:Cloud Functions may be spoofed by an attacker and this may lead to unauthorized access to Web Application. Consider using a standard authentication mechanism to identify the external entity.
Justification:Spoofed cloud functions have access to code snippets. Therefore they can read snippet data and change them.

5. Cross Site Scripting  [State: Mitigation Implemented]  [Priority: High] 

Category:Tampering
Description:The web server 'Web Application' could be a subject to a cross-site scripting attack because it does not sanitize untrusted input.
Justification:There is no other way to use CLOUD functions without specifingn CORS.

6. Elevation Using Impersonation  [State: Mitigation Implemented]  [Priority: High] 

Category:Elevation Of Privilege
Description:Web Application may be able to impersonate the context of Cloud Functions in order to gain additional privilege.
Justification:Cloud functions return only code that represent some snippet. There is no chance to get abny other snippet from cloud functions(except of brute force).

Interaction: WSS

7. Spoofing of Source Data Store Realtime Database  [State: Not Applicable]  [Priority: High] 

Category:Spoofing
Description:Realtime Database may be spoofed by an attacker and this may lead to incorrect data delivered to Cloud Functions. Consider using a standard authentication mechanism to identify the source data store.
Justification:Security is performed by Firebase

8. Weak Access Control for a Resource  [State: Not Applicable]  [Priority: High] 

Category:Information Disclosure
Description:Improper data protection of Realtime Database can allow an attacker to read information not intended for disclosure. Review authorization settings.
Justification:Security is performed by Firebase

Interaction: WSS

9. Spoofing of Destination Data Store Realtime Database  [State: Not Applicable]  [Priority: High] 

Category:Spoofing
Description:Realtime Database may be spoofed by an attacker and this may lead to data being written to the attacker's target instead of Realtime Database. Consider using a standard authentication mechanism to identify the destination data store.
Justification:Security is performed by Firebase.
